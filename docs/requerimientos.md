# Requerimientos del Sistema

## Descripción General

El Sistema de Reportes para AprendeLoPE tiene como finalidad centralizar la visualización de indicadores clave de desempeño (KPIs) relacionados con la gestión de canales digitales, permitiendo un monitoreo eficiente de métricas comerciales y operativas.

---

# Requerimientos Funcionales

## RF01 - Autenticación

El sistema debe permitir el acceso de usuarios autorizados mediante integración con Slack.

## RF02 - Visualización de KPIs

El sistema debe mostrar indicadores clave relacionados con la gestión comercial y digital.

## RF03 - Filtros Dinámicos

El sistema debe permitir filtrar la información mediante diferentes criterios seleccionables por el usuario.

## RF04 - Dashboard Centralizado

El sistema debe mostrar la información consolidada en un dashboard de fácil interpretación.

## RF05 - Consumo de API

El sistema debe consumir información desde servicios REST proporcionados por la empresa.

---

# Requerimientos No Funcionales

## RNF01

El sistema debe ser accesible desde navegadores modernos.

## RNF02

La interfaz debe ser responsiva para diferentes tamaños de pantalla.

## RNF03

La aplicación debe presentar tiempos de carga adecuados para la visualización de información.

## RNF04

La arquitectura debe permitir la incorporación futura de nuevos indicadores y reportes.

---

# KPIs Iniciales

* Porcentaje de ventas.
* Cantidad de ventas.
* Monto total vendido.
* Clientes interesados.
* Clientes en negociación.
* Pagos pendientes.
* Oportunidades perdidas.
* Tiempo promedio de conversión.
* Métricas de seguimiento de mensajes.
